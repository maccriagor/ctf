# Hash Collision Attacks

This guide introduces students to hash collision attacks using three practical tools: **HashClash**, **fastcoll**, and **SHAttered**.

---

## ✅ 1. HashClash (MD5 collisions with file control)

### 📌 Goal:

Generate two different files with the **same MD5 hash** using controlled prefixes.

### 🔧 Setup:

```bash
sudo apt install cmake g++ git
git clone https://github.com/cr-marcstevens/hashclash.git
cd hashclash
make
```

### 🚀 Run a Simple Collision:

```bash
cd md5collgen
./md5collgen --output1=msg1.bin --output2=msg2.bin
```

### 🧪 Verify:

```bash
md5sum msg1.bin msg2.bin
# Same hash output for both
```

### 📝 Educational Tip:

* Open the two files (`xxd msg1.bin`) to show they’re **different in content**, but **same MD5 hash**.
* Discuss the **structure of the MD5 compression function** and **differential paths**.

---

## ✅ 2. fastcoll (Fast MD5 collision tool)

### 📌 Goal:

Quickly generate two colliding 128-byte MD5 messages.

### 🔧 Setup:

```bash
git clone https://github.com/corkami/collisions.git
cd collisions/md5
make
```

### 🚀 Run Collision:

```bash
./fastcoll -o file1.bin file2.bin
```

### 🧪 Verify:

```bash
md5sum file1.bin file2.bin
# Identical MD5 hashes
```

### 📚 Teaching Notes:

* This shows how **fast** MD5 collisions can be generated on ordinary machines.
* Compare hex dumps using `xxd` or `cmp` to highlight exact byte differences.

## ✅ 3. SHAttered (First public SHA1 collision)

### 📌 Goal:

See a real-world **SHA1 collision** in action using PDFs.

### 📦 Download:

```bash
wget https://shattered.io/static/shattered-1.pdf
wget https://shattered.io/static/shattered-2.pdf
```

### 🧪 Verify:

```bash
sha1sum shattered-1.pdf shattered-2.pdf
# Same SHA1 hash

diff shattered-1.pdf shattered-2.pdf
# Files differ in content

md5sum shattered-1.pdf shattered-2.pdf
# Different MD5 hashes
```

### 📚 Teaching Notes:

* Explain **collision resistance** vs. **preimage resistance**.
* Show how digital signatures could be tricked using SHA1.
* Use a PDF viewer to show visual differences in the two files.

| --------- | --------- | ------------------ | --------- | ----------------- | ----------------------------- |
| HashClash | MD5       | Collision          | Medium    | High              | Controlled colliding files    |
| fastcoll  | MD5       | Collision          | Very Fast | None              | Quick collision generation    |
| SHAttered | SHA1      | Collision (static) | Pre-made  | None              | Demonstration, not modifiable |

---

## 🔐 Educational Outcome

By completing these exercises, students will understand:

* Why **MD5** and **SHA1** are insecure.
* The practicality of generating **hash collisions**.
* Real-world implications, such as **digital forgery**.
* The importance of using **secure hash algorithms** like **SHA-256**.
